HTML Fundamentals

Semantic HTML

Normal HTML tags like <div> and <span> are generic containers that don't convey any meaning about their content. They're purely for styling and layout.

Semantic HTML tags clearly describe their meaning and purpose to both the browser and the developer. For example, <header> tells you it contains header content, <article> indicates a self-contained piece of content.

Why it matters

  • Accessibility: Screen readers use semantic tags to help visually impaired users navigate pages. A screen reader can jump between <nav> sections or <article> elements, making navigation much easier.
  • SEO: Search engines better understand your content structure. A <main> tag signals the primary content, while <aside> indicates supplementary information.
  • Maintainability: Code becomes self-documenting. Instead of <div class = "header">, you use <header>, making your HTML clearer to other developers (and your future self).
  • Browser features: Browsers can provide built-in functionality based on semantic tags, like reader modes that extract <article> content.
